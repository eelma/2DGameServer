C_는 클라이언트 패킷

S_는 서버 패킷

회사에서 자주 사용하는 Jinja2 library

온라인 게임을 켜서 접속하면 클라이언트 쪽에서 로그인 요청

서버는 클라이언트가 건내준 정보를 토대로 성공여부 리턴

그 후 DB에서 해당 유저와 관련된 정보를 긁어서 같이 건내줌
플레이어의 정보는 Struct.proto에 만들어줌

Struct의 plyer의 id 는 플레이어의 수에따라 1씩 증가할것

S_LOGIN
생성한 플레이어가 여러개가 있을수도 있으니까 repeated로 여러개를 배열형태로 묶어서 보내줌

C_ENTER_GAME 
나 이 캐릭터로 게임에 들어갈래
플레이어 목록 중에 몇번째 플레이어인지

S_ENTER_GAME
서버쪽에서 접속이 가능한지 체크해서 성공 여부 반환
원래는 여기서 주변 플레이어의 정보등을 보내줘야하는데 지금은 스킵

S_CHAT은 해당하는 플레이어가 누구인지 메시지와 같이 전달

로그인 정보는 언제 호출이 돼야하느냐?
처음에 접속할때 호출이 돼야한다
원래는 여기서 비밀번호나 아이디를 입력해서 로그인하는 부분까지 처리돼야하는데
그건 게임서버에 바로 붙지 않고 외부서버랑 붙는다
